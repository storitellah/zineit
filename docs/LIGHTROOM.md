# ZineIt for Lightroom Classic

A plug-in that turns a Lightroom selection into a ready-to-lay-out ZineIt project:
your photos rendered with their develop settings, your captions carried across from
the catalogue, placed one per page at their true proportions.

## What it does — and what it deliberately doesn't

**It does**: adds an export destination (*ZineIt zine / photobook*) that renders the
selected photos, reads their IPTC caption / title / headline, builds a complete ZineIt
project as a single `.bak`, and opens ZineIt so you can start laying out.

**It doesn't**: run the ZineIt editor inside Lightroom. The Lightroom SDK has no
embedded browser — plug-in interfaces are built from Lua and `LrView`, which offers
text fields, pop-ups, and sliders, but no canvas and no drag-and-drop surface. A
layout editor cannot be faithfully rebuilt in it, and a cramped imitation would be
worse than the real thing. So Lightroom does what Lightroom is best at — selecting,
developing, and captioning — and hands off to ZineIt for layout.

Nothing is uploaded. The `.bak` is written to a folder you choose, on your machine.

## Install

1. Copy the `zineit.lrplugin` folder somewhere permanent (not your Downloads).
2. Lightroom Classic → **File ▸ Plug-in Manager ▸ Add** → select `zineit.lrplugin`.
3. It appears as *ZineIt by Storitellah*. Done — no restart needed.

Requires Lightroom Classic 6 / CC 2015 or newer (SDK 6.0+).

## Use

1. Select your photos in the Library, **in the order you want them to appear**
   (the export follows the current sort order).
2. **File ▸ Export**, and set *Export To:* **ZineIt zine / photobook**.
3. In the **ZineIt project** panel choose the format, the zine title, the margin,
   and where captions should come from. In **Project file**, pick a folder for the
   `.bak`.
4. Export. When it finishes, ZineIt opens — choose **Restore from .bak** and pick
   the file that was just written.

### Notes on choices the plug-in makes

- **Image sizing** defaults to 2400 px on the long edge — roughly 8 in at 300 ppi,
  the right ballpark for zine printing. Change it in the standard *Image Sizing*
  panel if you're printing larger; JPEG/sRGB are enforced because that's what the
  browser can render and what print shops expect.
- **One photo per page**, centred, at the photo's true aspect ratio — never stretched.
  Rearranging, resizing, and spreads are what ZineIt itself is for.
- **Mini zine** is a fixed 8-page format. Select more than it holds and the plug-in
  tells you exactly how many will be left out *before* rendering anything.
- **Captions** become caption-role text blocks beneath the photo, editable in ZineIt's
  Text Editor tab like any other text.
- **Thumbnails** aren't generated by the plug-in; ZineIt rebuilds its own preview and
  thumbnail tiers from the originals when the project is restored.

## How it's put together

```
zineit.lrplugin/
  Info.lua                          manifest: export service, menu items, version
  ZineItProject.lua                 formats, layout maths, project assembly, .bak writer
  ZineItJson.lua                    JSON encoder (the SDK ships none)
  ZineItExportServiceProvider.lua   export dialog + render/build/launch pipeline
  ZineItInfoProvider.lua            Plug-in Manager panel, feedback links
  ZineItMenuOpen.lua                Library ▸ Plug-in Extras ▸ Open ZineIt
  ZineItMenuFeedback.lua            Library ▸ Plug-in Extras ▸ Report a bug or idea
```

Two design decisions worth knowing:

- **`ZineItProject` and `ZineItJson` import nothing from Lightroom.** That keeps the
  layout maths and file format testable on plain Lua — which is most of the logic and
  all of the risk.
- **The `.bak` is streamed, not built in memory.** The project JSON is encoded, its
  closing brace trimmed, and each photo base64-encoded and appended one at a time, so
  peak memory stays at roughly one photo regardless of how many you export.

## Testing

```bash
lua5.4 lightroom/tests/run-lua-tests.lua      # 22 unit tests: JSON, layout, .bak writer
lua5.4 lightroom/tests/make-fixture.lua       # regenerate the contract fixture
cd tests && npm test                          # 97 tests, incl. the ZineIt contract
```

The contract tests are the important ones: a `.bak` generated by the plug-in's own Lua
code is fed to the **real** ZineIt, restored through its real restore path, and checked
— pages, captions, cover title, stored photo blobs, print DOM, imposition. That's what
proves the two halves agree.

### What automated testing here cannot prove

The Lightroom-facing files (`Info.lua`, the export provider, the menu items) are
syntax-checked and hold no untested logic, but they call Adobe's APIs — which exist
only inside Lightroom. Specifically unverified until run there:

- `LrStringUtils.encodeBase64` behaving like a standard encoder (the fixture uses a
  reference implementation verified byte-for-byte against Python's decoder);
- `photo:getRawMetadata('croppedDimensions')` and `getFormattedMetadata(...)` field names;
- export dialog rendering and `processRenderedPhotos` rendition flow.

**Please run it against a real catalogue before relying on it.** Bugs and ideas:
**hello@storitellah.com** — there's a menu item and a Plug-in Manager button that
write the email for you.
